# 04 沙箱与工具治理设计

## 为什么沙箱是核心

Coding Agent 和普通聊天 Agent 最大区别在于：

```text
它会接触真实文件系统，甚至未来会执行命令。
```

所以必须先设计沙箱。

不要先写工具再补安全。

正确顺序：

```text
Workspace root
  -> Path normalization
  -> Sandbox policy
  -> Tool permission decision
  -> User confirmation
  -> Tool execution
  -> Audit log
```

## Workspace Root

每个会话必须绑定一个 workspace。

```text
Workspace
- id
- name
- rootPath
- status
- createdAt
- updatedAt
```

rootPath 必须是绝对路径。

用户输入的任何相对路径都要基于 rootPath 解析。

## 路径安全

所有文件工具必须走路径归一化。

规则：

```text
1. 把用户输入路径解析成绝对路径
2. normalize
3. resolve symbolic path if needed
4. 确认最终路径仍在 workspace root 内
5. 禁止访问 root 外路径
```

必须防止：

```text
../
..\
绝对路径逃逸
符号链接逃逸
隐藏路径绕过
大小写绕过
```

第一版可以先禁止 follow symlink。

## 工具风险分级

建议风险等级：

```text
LOW
  list_files
  read_file
  search_code

MEDIUM
  propose_patch
  git_diff
  git_status

HIGH
  apply_patch
  write_file
  run_tests

CRITICAL
  run_command
  delete_file
  move_file
  git_commit
  network_call
```

第一版只开放 LOW 和 propose_patch。

## 第一版工具

### list_files

输入：

```text
path
maxDepth
```

输出：

```text
files/directories
size
modifiedAt
isDirectory
```

安全：

```text
必须在 workspace 内
限制最大返回数量
忽略 .git / node_modules / target 等大目录，可配置
```

### read_file

输入：

```text
path
maxChars
```

输出：

```text
content
truncated
```

安全：

```text
只读
限制最大文件大小
限制二进制文件
限制敏感文件
```

默认拒绝：

```text
.env
*.pem
*.key
id_rsa
credentials
secrets
```

### search_code

输入：

```text
query
path
fileGlob
maxResults
```

实现可先用 ripgrep。

安全：

```text
只在 workspace 内搜索
限制结果数量
限制单条内容长度
```

### propose_patch

输入：

```text
unifiedDiff
summary
```

输出：

```text
patchId
filesChanged
riskLevel
requiresApproval=true
```

这个工具不直接写文件，只生成待审核 patch。

### apply_patch_after_confirm

第一版可以先不开放给模型直接调用。

由前端用户确认后，通过后端 API 调用。

流程：

```text
patch proposal
  -> sandbox validates paths
  -> frontend shows diff
  -> user approves
  -> backend applies patch
  -> audit log
```

## 用户确认模型

工具确认分三类：

```text
ToolConfirmationRequired
  工具执行前确认

PatchApprovalRequired
  diff 应用前确认

DecisionRequired
  多方案选择或需要用户补充信息
```

不要把所有确认都混成一种。

## 命令执行治理

第二阶段再做 `run_command`。

必须有：

```text
command allowlist
working directory check
timeout
output size limit
no interactive command
no destructive command by default
user confirmation
```

第一批允许命令可以是：

```text
mvn test
mvn -q -DskipTests compile
npm run build
npm test
```

不允许：

```text
rm
rmdir
del
format
shutdown
git reset --hard
git clean
curl | sh
powershell -EncodedCommand
```

## 审计日志

每次工具调用都应该记录：

```text
toolName
arguments
workspaceId
riskLevel
decision
startedAt
endedAt
durationMs
resultSummary
error
```

高风险工具必须记录用户确认人和确认时间。

## 沙箱第一原则

```text
模型只是提出动作，平台决定动作能不能执行。
```

不要相信模型自己说“这是安全的”。
