# 06 实施路线图

## 阶段 0：项目骨架

目标：能启动后端，保留清晰模块边界。

建议技术：

```text
Java 17+
Spring Boot 3.x
Maven
AgentScope 2.0.0-RC1
SSE via Spring MVC SseEmitter
```

建议目录：

```text
src/main/java/.../adapter/web
src/main/java/.../application
src/main/java/.../domain
src/main/java/.../infrastructure/agentscope
src/main/java/.../infrastructure/workspace
src/main/java/.../infrastructure/sandbox
src/main/java/.../infrastructure/memory
```

先不要做复杂多模块 Maven，除非后续边界稳定。

## 阶段 1：Workspace + Read-only Tools

目标：Agent 能安全读取项目。

实现：

```text
WorkspaceController
WorkspaceApplicationService
WorkspaceRepository
SandboxPathResolver
ListFilesTool
ReadFileTool
SearchCodeTool
```

API：

```text
POST /api/workspaces
GET  /api/workspaces
GET  /api/workspaces/{id}/tree
```

Agent tools：

```text
list_files
read_file
search_code
```

验收：

```text
不能读取 workspace 外路径
不能读取敏感文件
搜索结果有限制
大文件会截断
```

## 阶段 2：AgentScope Runtime + SSE

目标：网页能看到 Agent 运行过程。

实现：

```text
AgentRuntimePort
AgentScopeRuntimeAdapter
AgentStreamEvent
ChatController /stream
AgentScopeExecutionTrace
```

事件：

```text
phase
timing
model
tool
delta
final
error
```

验收：

```text
模型输出能流式显示
工具调用能展示
模型调用耗时能展示
Agent 总耗时能展示
```

## 阶段 3：Patch Proposal

目标：Agent 可以提出代码修改，但不能直接写文件。

工具：

```text
propose_patch
```

流程：

```text
Agent 生成 unified diff
后端校验 diff 涉及路径
保存 patch proposal
前端展示 diff
等待用户确认
```

验收：

```text
没有用户确认不能写文件
diff 涉及 workspace 外路径必须拒绝
diff 涉及敏感文件必须拒绝
```

## 阶段 4：Apply Patch After Confirm

目标：用户确认后应用 patch。

实现：

```text
PatchApprovalController
PatchApplicationService
PatchApplier
PatchAuditLog
```

验收：

```text
确认后才写文件
写入前再次 sandbox 校验
写入后返回文件列表和摘要
保留审计日志
```

## 阶段 5：Memory MVP

目标：记忆能影响 Coding Agent。

实现：

```text
MemoryEntry
MemoryApplicationService
LongTermMemoryExtractor
MemoryController
MemoryContextAssembler
```

记忆来源：

```text
用户明确要求记住
用户设置项目约束
工具验证后的项目事实
```

验收：

```text
active 记忆注入 Agent
candidate 不注入
conflict 不注入
用户可 approve / reject / disable
```

## 阶段 6：Sandbox Governance

目标：高风险动作进入确认流程。

实现：

```text
ToolRiskClassifier
ToolConfirmationRequired
DecisionRequired
SandboxAuditLog
```

工具风险：

```text
read: auto allow
patch proposal: allow but not apply
apply patch: require confirm
run command: require confirm and allowlist
```

## 阶段 7：MiddlewareBase 治理

目标：把 AgentScope 的运行治理系统化。

研究并实现：

```text
onAgent
onReasoning
onActing
onModelCall
onSystemPrompt
```

用途：

```text
模型调用超时
模型调用重试
工具权限检查
动态 system prompt
上下文预算控制
```

## 阶段 8：命令执行

目标：允许受控命令。

先允许：

```text
mvn test
mvn -q -DskipTests compile
npm run build
npm test
```

必须具备：

```text
allowlist
cwd must be workspace root or child
absolute path check
timeout
output limit
user confirmation
```

## 推荐第一天任务

第一天不要做全套。

只做：

```text
1. Spring Boot 项目骨架
2. AgentScope 依赖
3. Workspace model
4. SandboxPathResolver
5. list_files/read_file/search_code 三个 read-only 工具
6. 一个 /api/chat/stream 验证 AgentScope SSE
```

成功以后，再做 patch proposal。
