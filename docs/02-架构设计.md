# 02 架构设计

## 总体架构

推荐第一版用 Spring Boot 单体后端，前端后续用 Vue 或 React。

后端分层：

```text
adapter-web
  REST / SSE / Web API

application
  用例编排：会话、Agent 调用、工具确认、记忆注入

domain
  Workspace、SandboxPolicy、ToolCall、Memory、AgentSession 等核心模型

infrastructure
  文件系统、AgentScope、数据库、搜索、patch 应用等外部适配
```

不要一上来拆太多服务。

## 核心模块

### Workspace 模块

负责：

- 注册 workspace root。
- 校验路径是否在 workspace 内。
- 保存 workspace 元数据。
- 提供目录树、文件摘要。

核心对象：

```text
Workspace
WorkspaceId
WorkspaceRoot
WorkspacePolicy
```

### Sandbox 模块

负责：

- 路径归一化。
- 禁止路径逃逸。
- 控制可读、可写、可执行范围。
- 决定工具调用是否需要用户确认。

核心对象：

```text
SandboxPolicy
PathAccessDecision
ToolPermissionDecision
RiskLevel
```

### Tool 模块

第一版工具：

```text
list_files
read_file
search_code
propose_patch
apply_patch_after_confirm
```

第二版工具，命令级沙箱已完成第一轮：

```text
Bash
Shell
run_command
git_status via Bash
git_diff via Bash
run_tests via Bash
```

### Agent Runtime 模块

负责：

- 构建 AgentScope ReActAgent。
- 注册工具。
- 注入系统 prompt 和记忆。
- 监听 streamEvents。
- 转换成平台自己的 runtime event。
- 对接 SSE。

核心接口：

```text
AgentRuntimePort
AgentRuntimeAdapter
AgentStreamEvent
```

### Memory 模块

负责：

- 用户偏好。
- 项目事实。
- 项目约束。
- 技能记忆。
- 记忆审核。
- 冲突处理。
- 上下文注入。

记忆不要写死在 AgentScope 内部。

### Confirmation 模块

负责：

- 工具调用前确认。
- 多方案选择。
- diff 审核。
- 外部执行结果回填。

第一版可以用平台自己的事件：

```text
DecisionRequired
ToolConfirmationRequired
PatchApprovalRequired
```

AgentScope 原生的 `RequireUserConfirmEvent` 可用于工具确认，但不要把所有业务选择都硬塞进去。

## 请求链路

一次用户请求推荐链路：

```text
ChatController
  -> AgentApplicationService
      -> load session
      -> load workspace
      -> build memory context
      -> build sandbox context
      -> AgentRuntimePort.executeStreaming
          -> AgentScope ReActAgent
          -> tools
          -> streamEvents
      -> persist messages
      -> capture memory
      -> return final reply
```

## SSE 链路

```text
AgentScope Flux<AgentEvent>
  -> doOnNext trace recorder
  -> AgentStreamEvent
  -> SseEmitter
  -> frontend fetch stream
  -> runtime panel / chat bubble
```

第一版可以继续使用 Spring MVC + `SseEmitter`。

不需要一开始切 WebFlux。

## 数据存储建议

第一版可以用数据库保存：

```text
workspace
agent_session
agent_message
agent_execution
tool_call
memory_entry
confirmation_request
```

也可以先用本地文件或 H2 快速验证。

但 sandbox policy 和 memory 最好从一开始就是结构化模型，不要只塞 JSON 文本。

## 关键边界

### AgentScope 不是平台边界

AgentScope 是 runtime，不是整个平台。

平台能力包括：

```text
workspace 管理
sandbox 治理
用户确认
记忆生命周期
审计日志
前端交互
```

这些必须由我们自己的代码控制。

### Tool 不能直接等于权限

注册了工具，不代表工具可以随便执行。

每次工具调用都要经过：

```text
参数解析
路径校验
风险判断
用户确认
执行
审计
```

### 记忆不能直接等于 prompt

记忆有自己的状态和审核流程。

注入 prompt 的只能是经过筛选的 active memory。
