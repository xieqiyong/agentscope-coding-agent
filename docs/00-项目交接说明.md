## 背景

本项目从 `ragmem` 学习项目中拆出。

`ragmem` 已经完成了这些学习和实验：

- RAG 知识库上传、解析、切分、向量化、Milvus 检索。
- AgentScope Java 接入。
- AgentScope `ReActAgent` 与知识库工具打通。
- SSE 流式返回。
- AgentScope `streamEvents` 事件观测。
- 短期记忆、长期记忆候选、自动生效、冲突处理。
- Agent 运行阶段耗时统计。
- 初步理解 AgentScope deprecated Hook 和新的 `MiddlewareBase` 方向。

新项目不继续承载 RAG 学习页面，而是聚焦一个新的产品主线：

```text
基于 Java AgentScope 的网页版 Coding Agent。
```

目标类似 opencode 的体验，但不是复刻它，而是做一个适合 Java 技术栈、可学习、可扩展、带记忆和沙箱治理的 Coding Agent 平台。

## 新项目目标

第一阶段目标是做一个可工作的 MVP：

```text
用户在网页输入开发任务
  -> Agent 读取工作区文件
  -> Agent 搜索代码
  -> Agent 生成修改建议
  -> 前端展示 diff
  -> 用户确认后才允许应用修改
```

第一版不急着执行 shell 命令，不急着自动改文件。

先做好：

- workspace 边界
- 文件读取工具
- 搜索工具
- patch 提议工具
- SSE 事件流
- 用户确认模型
- 记忆入口

## 下一个 Agent 应该先做什么

1. 读取根目录 `AGENTS.md`。
2. 读取本目录全部文档。
3. 不要直接开始写业务代码。
4. 先确认 MVP 边界：读文件、搜代码、生成 diff、用户确认。
5. 先设计 sandbox，再设计工具。
6. 先做 read-only 工具，再做 write 工具。
